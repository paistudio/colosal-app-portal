# Proposals Page — Design

## Purpose

New `/proposals` route (nav link already exists in `AppSidebar`, unused). Lets user
review AI-generated Upwork proposals (cover letter + question answers) and manage
reusable proposal templates.

## Data

- `public.proposals` (8 rows) — `user_id`, `upwork_job_id` (FK → `upwork_jobs.id`,
  nullable), `attachment` (uuid[]), `client_paint`, `cover_letter_generated`,
  `cover_letter_template`, `is_submitted`, `last_generated_at` (date),
  `question_answer` (jsonb), `question_instruction`, `total_token_used`.
- `public.proposal_templates` (4 rows) — `user_id`, `name`, `template`,
  `ai_instruction`. Referenced by `user_scan_config.proposal_template_id`
  (nullable FK, no cascade).
- `public.upwork_jobs` — joined for job title/url/budget/scan_config_id.
- RLS enabled on both tables; server client scopes to `auth.uid()` automatically
  same as existing `job-scanner` queries (no explicit `.eq("user_id", ...)` used
  elsewhere in repo — RLS policy handles it).

## Routes / Files

```
app/(dashboard)/proposals/page.tsx              server component, tab switch via ?tab=
app/(dashboard)/proposals/proposal-list.tsx      client, proposals tab list
app/(dashboard)/proposals/[id]/page.tsx          server, proposal detail
app/(dashboard)/proposals/[id]/proposal-detail.tsx  client, copy-to-clipboard view
app/(dashboard)/proposals/template-list.tsx      client, templates tab list + delete
app/(dashboard)/proposals/template-form.tsx      client, create/edit dialog
```

## Tab 1 — Proposals

`page.tsx` (default tab) fetches:

```sql
proposals.* joined with upwork_jobs(title, url, amount_display, client_country, scan_config_id)
order by inserted_at desc
```

`ProposalList` renders `ScannerList`-style rows (rounded-3xl card, ring, hover):
- Badge: "Submitted" (emerald) if `is_submitted`, else "Draft" (amber) — reuse
  `STATUS_STYLES`-like map, new `PROPOSAL_STATUS_STYLES` in a shared format file.
- Job title (truncate), client country if present.
- `last_generated_at` formatted (date, not relative — column is `date` type).
- Click row → `/proposals/[id]`. No inline edit/delete actions on this tab.
- Empty state: dashed border, "No proposals yet." (no CTA — proposals are
  generated by the scan pipeline, not created here).

## Proposal Detail (`/proposals/[id]`)

Server component fetches the proposal + joined job by id (404 via `notFound()`
if missing or not owned — RLS returns no row).

Layout: back link to `/proposals`, job card at top (title linking to Upwork URL,
budget, "View scanner" link to `/job-scanner/{scan_config_id}` if present).

Below: `ProposalDetail` client component —
- **Cover Letter** section: `cover_letter_generated` in a read-only styled block
  + Copy button (`navigator.clipboard.writeText`, `toast.success("Copied")`).
- **Question Answers** section: `question_answer` jsonb rendered as list of
  Q/A pairs (assume `{question: string, answer: string}[]` shape, matching
  `scan_config.qa_example` convention), each with its own Copy button.
- No edit/regenerate here — read/copy only, per requirements.

## Tab 2 — Templates

`TemplateList` (client) fetches `proposal_templates` for the user, renders cards:
name, truncated `template` preview (first ~120 chars), edit (pencil) and delete
(trash) icon buttons — same icon-button row as `ScannerList`.

**Create/Edit** — `TemplateForm` opens in a `Dialog` (shadcn) with fields:
`name` (text input), `template` (textarea, helper text explains
`[prompt]…[/prompt]` marks AI-filled sections), `ai_instruction` (textarea).
Upsert via `supabase.from("proposal_templates").insert/update`.

**Delete** — before deleting, query
`user_scan_config.select("id", { count: "exact", head: true }).eq("proposal_template_id", id)`.
If count > 0, `toast.error("Template in use by N scanner(s)")` and abort.
Otherwise delete and `router.refresh()`.

Empty state: dashed border, "No templates yet." + "Create template" button
opening the dialog.

## Out of scope

- Regenerating/editing `cover_letter_generated` or `question_answer` from the UI.
- Submitting proposals to Upwork from this page (`is_submitted` is set elsewhere).
- Pagination (row counts are tiny: 8 proposals, 4 templates today).
